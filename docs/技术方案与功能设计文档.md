## 1. 项目概述

### 1.1 项目背景

随着网络环境的复杂化，用户对网络代理工具的需求日益增长。传统的代理软件普遍存在以下问题：

- **跨平台兼容性差**：多数软件仅支持单一操作系统
- **界面交互复杂**：功能堆砌，用户体验欠佳
- **性能开销大**：资源占用高，运行效率低
- **扩展性有限**：难以自定义和二次开发

本项目旨在开发一款轻量、高效、易用的跨平台代理软件，基于 MetaCubeX/mihomo 内核，提供流畅的用户体验和强大的功能支持。

### 1.2 项目目标

**核心目标**：

- 构建一款基于 Mihomo 内核的跨平台（Windows/macOS/Linux）代理软件
- 提供简洁直观的用户界面，参考现代设计趋势
- 支持服务商（订阅）管理、节点选择、延迟测试等核心功能
- 实现轻量级部署和高性能运行

**功能目标**：

- ✅ 服务商订阅管理（添加、编辑、删除、刷新）
- ✅ 节点选择与切换（支持延迟显示）
- ✅ 一键测速功能
- ✅ 连接状态实时监控
- ✅ 流量统计与显示
- ✅ 浅色/深色主题切换
- ✅ 系统代理配置

### 1.3 目标用户

| 用户类型 | 使用场景 | 核心需求 |
|---------|---------|---------|
| 技术人员 | 开发测试、网络调试 | 高可配置性、性能稳定 |
| 普通用户 | 日常上网、隐私保护 | 简单易用、稳定可靠 |
| 企业用户 | 团队协作、安全办公 | 多账号管理、分组控制 |

### 2.2 技术选型理由

#### 2.2.1 桌面框架：Tauri 2.0

| 特性 | Tauri 2.0 | Electron | 对比说明 |
|------|----------|----------|---------|
| 体积 | 2-10 MB | 80-150 MB | Tauri 体积小 10-50 倍 |
| 内存占用 | 30-50 MB | 150-300 MB | Tauri 节省 70% 内存 |
| 启动速度 | < 100ms | 2-5s | Tauri 启动更快 |
| 安全性 | Rust 沙箱 | Node.js 沙箱 | Tauri 安全性更高 |
| 生态 | 快速发展中 | 成熟稳定 | Electron 生态更成熟 |

**选择理由**：

- 轻量级设计，适合资源敏感场景
- Rust 后端提供高性能和安全性
- 支持现代 Web 技术栈（React/Vue）
- 内置系统集成能力（系统托盘、通知等）

#### 2.2.2 前端框架：React 18

**选择理由**：

- 组件化开发模式，代码复用性高
- 虚拟 DOM 机制，性能优秀
- 丰富的生态系统和社区支持
- TypeScript 支持，类型安全

#### 2.2.3 UI 组件库：Shadcn UI

**选择理由**：

- 基于 Radix UI 的无障碍组件
- Tailwind CSS 驱动，样式灵活
- 代码可控，易于定制
- 现代化设计语言

#### 2.2.4 状态管理：Zustand

**选择理由**：

- 轻量级（无 Provider 嵌套）
- TypeScript 友好
- Redux DevTools 支持
- 简单的 API 设计

### 3.5 延迟测试模块

#### 3.5.1 测试策略

| 方式 | 描述 | 优缺点 |
|------|------|--------|
| TCP 连接 | 测试 TCP 握手时间 | 准确但耗时 |
| HTTP HEAD | 测试 HTTP 响应时间 | 包含网络延迟 |
| ICMP Ping | 测试 ICMP 延迟 | 可能被防火墙拦截 |

**本项目采用 TCP 连接方式**，综合考虑准确性和通用性。

#### 3.5.2 延迟显示规则

| 延迟范围 | 颜色编码 | 用户感知 |
|---------|---------|---------|
| < 100ms | 绿色 | 优秀 |
| 100-300ms | 黄色 | 良好 |
| > 300ms | 红色 | 较差 |
| 超时 | 灰色 | 不可用 |

## 4. 数据结构设计

### 4.1 前端数据类型（TypeScript）

```typescript
// 服务商（订阅）类型
interface Provider {
  id: string;                    // UUID
  name: string;                  // 服务商名称
  url: string;                   // 订阅链接
  group?: string;                // 分组名称
  enabled: boolean;              // 是否启用
  lastUpdated: string;           // 最后更新时间 (ISO 8601)
  nodeCount: number;             // 节点数量
}

// 节点类型
interface Node {
  id: string;                    // UUID
  name: string;                  // 节点名称
  providerId: string;            // 所属服务商 ID
  type: NodeType;                // 节点类型
  server: string;                // 服务器地址
  port: number;                  // 端口
  delay?: number;                // 延迟 (ms)
  enabled: boolean;              // 是否启用
}

type NodeType = 'http' | 'socks5' | 'vmess' | 'trojan' | 'ss' | 'vless';

// 代理状态类型
interface ProxyStatus {
  isConnected: boolean;          // 连接状态
  currentNode?: Node;            // 当前节点
  uploadBytes: number;           // 上传字节数
  downloadBytes: number;        // 下载字节数
  latency?: number;              // 当前延迟
}

// 应用配置类型
interface AppConfig {
  theme: 'light' | 'dark';       // 主题
  providers: Provider[];          // 服务商列表
  nodes: Node[];                 // 节点列表
  proxy: {
    listen: string;              // 监听地址
    httpPort: number;            // HTTP 端口
    socks5Port: number;          // SOCKS5 端口
  };
  startup: {
    autoStart: boolean;          // 开机自启
    autoConnect: boolean;       // 自动连接
  };
}
```

### 4.2 后端数据结构（Rust）

```rust
// 服务商结构
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Provider {
    pub id: String,
    pub name: String,
    pub url: String,
    pub group: Option<String>,
    pub enabled: bool,
    pub last_updated: String,
    pub node_count: usize,
}

// 节点结构
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Node {
    pub id: String,
    pub name: String,
    pub provider_id: String,
    pub node_type: String,
    pub server: String,
    pub port: u16,
    pub delay: Option<u32>,
    pub enabled: bool,
}

// 代理配置
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProxyConfig {
    pub listen: String,
    pub http_port: u16,
    pub socks5_port: u16,
}

// 代理状态
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProxyStatus {
    pub is_running: bool,
    pub current_node: Option<String>,
    pub upload_bytes: u64,
    pub download_bytes: u64,
}

// 延迟测试结果
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct LatencyResult {
    pub node_id: String,
    pub delay: Option<u32>,
    pub error: Option<String>,
}
```

## 6. API 接口设计

### 6.1 接口概述

| 类型 | 说明 |
|------|------|
| Tauri Command | Rust 后端暴露给前端的命令 |
| 采用 invoke 机制 | 异步调用，返回 Promise |

### 6.2 代理控制接口

#### 6.2.1 启动代理

```typescript
invoke<string>('start_proxy')
// 返回: "Proxy started successfully"
```

#### 6.2.2 停止代理

```typescript
invoke<string>('stop_proxy')
// 返回: "Proxy stopped successfully"
```

#### 6.2.3 获取代理状态

```typescript
invoke<ProxyStatus>('get_proxy_status')
// 返回: ProxyStatus 对象
```

#### 6.2.4 设置当前节点

```typescript
invoke<void>('set_current_node', { nodeName: string })
```

### 6.3 服务商管理接口

#### 6.3.1 获取服务商列表

```typescript
invoke<Provider[]>('get_providers')
// 返回: Provider[] 数组
```

#### 6.3.2 添加服务商

```typescript
invoke<void>('add_provider', { provider: Provider })
```

#### 6.3.3 更新服务商

```typescript
invoke<void>('update_provider', { 
  id: string, 
  updates: Provider 
})
```

#### 6.3.4 删除服务商

```typescript
invoke<void>('delete_provider', { id: string })
```

#### 6.3.5 拉取订阅

```typescript
invoke<string>('fetch_subscription', { url: string })
// 返回: 订阅内容 (Base64 编码)
```

### 6.4 节点管理接口

#### 6.4.1 获取节点列表

```typescript
invoke<Node[]>('get_nodes')
// 返回: Node[] 数组
```

#### 6.4.2 按服务商获取节点

```typescript
invoke<Node[]>('get_nodes_by_provider', { providerId: string })
```

#### 6.4.3 测试节点延迟

```typescript
invoke<LatencyResult>('test_node_latency', {
  nodeId: string,
  server: string,
  port: number
})
```

#### 6.4.4 测试所有节点延迟

```typescript
invoke<LatencyResult[]>('test_all_nodes_latency')
```

### 6.5 配置管理接口

#### 6.5.1 获取代理配置

```typescript
invoke<ProxyConfig>('get_proxy_config')
```

#### 6.5.2 更新代理配置

```typescript
invoke<void>('update_proxy_config', { config: ProxyConfig })
```

### 6.6 错误处理

所有接口遵循统一的错误处理模式：

```typescript
try {
  const result = await invoke<T>('command_name', params);
  // 处理成功结果
} catch (error) {
  console.error('命令执行失败:', error);
  // 处理错误：显示提示信息
}
```
